# Arkanoid 개발 컨텍스트 문서 (Context.md)

이 문서는 Python/PyGame 기반 Arkanoid 클론 프로젝트의 핵심 개발 정보를 요약하여, 개발 과정 전반에 걸쳐 빠른 참고 자료로 활용하는 것을 목적으로 합니다.

## 1. 프로젝트 개요

- **목표**: Taito의 1986년작 Arkanoid를 재현하며 2D 게임 물리, AI 활용 개발, 빠른 프로토타이핑 역량 확보.
- **장르**: 벽돌깨기 (Block Breaker)
- **개발 환경**: Python 3.8+, PyGame 2.x
- **핵심 특징**: 파워업 시스템, 다양한 벽돌, 적 유닛, 최종 보스전.

## 2. 핵심 게임플레이 메커니즘

- **게임 목표**: 패들(Vaus)을 조종하여 공을 튕겨내고, 화면 상단의 모든 파괴 가능한 벽돌을 제거하여 스테이지를 클리어하는 것.
- **조작**:
  - **주 조작**: 마우스를 이용한 좌우 이동 (부드러운 아날로그 조작감).
  - **보조 조작**: 키보드 좌/우 화살표.
- **게임 오버**: 공을 패들로 받지 못하고 화면 하단으로 떨어뜨리면 라이프 1 감소. 모든 라이프 소진 시 게임 오버.
- **스테이지 클리어**: 파괴 가능한 벽돌(일반, 은색)을 모두 제거하면 스테이지 클리어. 다음 스테이지로 이동.

## 3. 게임 요소 상세

### 3.1. 패들 (Vaus)
- 플레이어가 조종하는 우주선.
- 공을 튕겨내는 역할을 하며, 패들의 어느 부분에 맞느냐에 따라 공의 반사 각도가 달라질 수 있음.

### 3.2. 공
- 일정한 속도로 움직이며 벽, 벽돌, 패들에 닿으면 물리 법칙에 따라 반사됨.
- 특정 파워업(Slow)에 의해 속도가 변할 수 있음.

### 3.3. 벽돌
- **일반 벽돌**: 1회 타격 시 파괴됨.
- **은색 벽돌**: 여러 번 타격해야 파괴됨 (스테이지에 따라 요구 타격 횟수 증가).
- **금색 벽돌**: 파괴 불가능. 영구적인 장애물.

### 3.4. 파워업 (캡슐)
벽돌 파괴 시 일정 확률로 드롭되며, 패들로 획득 시 특수 효과 발동.
- **S (Slow)**: 공의 속도를 일시적으로 감소시킴.
- **E (Enlarge)**: 패들의 길이가 일시적으로 늘어남.
- **L (Laser)**: 패들 양쪽에서 레이저를 발사하여 벽돌을 파괴할 수 있음.
- **C (Catch)**: 공이 패들에 닿았을 때 바로 튕겨나가지 않고, 플레이어가 원할 때 다시 발사할 수 있음.
- **D (Disrupt)**: 공이 3개로 분열됨 (멀티볼).
- **B (Break)**: 다음 스테이지로 바로 이동하는 출구가 열림.
- **P (Player)**: 추가 라이프 1개를 획득.

### 3.5. 적 유닛
- 화면 상단에서 나타나 불규칙하게 움직이며 공의 궤적을 방해.
- 공과 충돌 시 파괴되며 보너스 점수 획득.

### 3.6. 최종 보스 (DOH)
- 33번째 스테이지에 등장하는 최종 보스.
- 입 모양의 거대한 형태로, 16회 타격 시 격파 가능.
- 플레이어를 향해 폭탄을 발사하는 등 공격 패턴을 가짐.

## 4. 개발 마일스톤 요약

- **Phase 1**: 기본 물리 엔진 (공-벽/패들 충돌 및 반사), 패들 조작.
- **Phase 2**: 다양한 벽돌 타입, 스테이지 데이터 구조 및 레벨 로딩.
- **Phase 3**: 7종 파워업 시스템, 적 유닛 구현.
- **Phase 4**: UI(점수, 라이프), 사운드, 최종 보스 DOH 구현.
- **Phase 5**: 테스트, 밸런싱, 배포.

## 5. 주요 고려사항

- **저작권**: "Arkanoid" 명칭 및 원작의 그래픽/사운드 에셋 사용 불가. 프로젝트명은 변경하고, 모든 시각/청각 자료는 자체 제작하거나 저작권 문제가 없는 에셋을 사용해야 함.
- **물리 엔진**: PyGame의 내장 충돌 감지 함수를 최대한 활용하되, 정교한 반사 각도 계산 로직이 필요.
- **스테이지 디자인**: 초기 5~10개 스테이지로 시작. 레벨 디자인을 쉽게 할 수 있는 데이터 구조(JSON 등) 설계가 중요.
