# Arkanoid 게임 개발 기획서

## 1. 개발 목적

- **클래식 게임 재현 학습**: Taito의 1986년 명작 Arkanoid를 Python/PyGame으로 구현하며 벽돌깨기(Block Breaker) 장르의 핵심 메커니즘 이해
- **게임 물리 엔진 경험**: 공 반사 각도, 충돌 감지, 패들 조작 등 2D 물리 구현 역량 확보
- **AI 활용 개발 검증**: 생성형 AI를 통한 게임 로직 생성 및 바이브코딩으로 빠른 프로토타입 제작 가능성 실험
- **포트폴리오 다각화**: Minesweeper와 다른 장르(액션/반응속도)로 개발 역량 스펙트럼 확대

**참고**: Arkanoid는 1986년 Taito에서 개발한 아케이드 게임으로, Atari의 Breakout(1976)을 발전시킨 작품임. 특히 파워업 시스템, 다양한 벽돌 종류, 적 유닛 등을 도입해 장르를 혁신했으며, 1987년 일본 아케이드 최고 수익 게임으로 기록됨

## 2. 주요 기능 및 제공 가치

### 핵심 게임플레이
- **패들 조작**: Vaus(패들형 우주선)를 좌우로 이동하여 공 반사
- **벽돌 파괴 시스템**: 
  - 일반 벽돌 (1회 타격으로 파괴)
  - 은색 벽돌 (2회 이상 타격 필요, 스테이지마다 증가)
  - 금색 벽돌 (파괴 불가능)
- **파워업 시스템**: 벽돌 파괴 시 캡슐 드롭. 주요 파워업으로는 Slow(공 속도 감소), Enlarge(패들 확장), Laser(레이저 발사), Catch(공 포획), Disrupt(멀티볼), Break(스테이지 워프), Player(추가 생명) 등이 있음
- **적 유닛**: 화면에 등장하여 공 궤적 교란 (충돌 시 보너스 점수)
- **최종 보스**: 33번째 스테이지에서 DOH라는 보스와 대결 (16회 타격으로 격파)

### 제공 가치
- **반응속도 향상**: 실시간 공 추적 및 패들 위치 조정 훈련
- **전략적 플레이**: 파워업 선택적 획득, 벽돌 파괴 순서 결정 등 전술적 요소
- **레트로 감성**: 1980년대 아케이드 감성을 Python으로 재해석한 학습 자료

## 3. 개발 단계별 마일스톤

### Phase 1: 기본 물리 및 충돌 시스템 (1~2주차)
- 공 이동 및 벽면/패들 반사 로직 (각도 계산 포함)
- 패들-마우스/키보드 입력 연동 (아날로그 조작감 구현)
- 벽돌-공 충돌 감지 및 파괴 처리
- 게임 오버 조건 (공 하단 이탈)

### Phase 2: 벽돌 시스템 및 스테이지 (3주차)
- 3종 벽돌 타입 구현 (일반/은색/금색)
- 스테이지 레이아웃 데이터 구조 설계 (JSON 또는 2D 배열)
- 최소 5~10개 스테이지 패턴 제작
- 스테이지 클리어 판정 및 진행 시스템

### Phase 3: 파워업 및 적 유닛 (4주차)
- 파워업 캡슐 드롭 및 낙하 애니메이션
- 7종 파워업 효과 구현 (S/E/L/C/D/B/P)
- 적 유닛 생성 및 이동 패턴
- 파워업 중첩 로직 (일부 파워업은 상호 배타적)

### Phase 4: UI/사운드 및 최종 보스 (5주차)
- 점수, 생명, 스테이지 번호 UI
- 간단한 사운드 효과 (벽돌 파괴, 파워업 획득 등)
- DOH 보스 구현 (폭탄 발사 패턴 포함)
- 게임 오버/클리어 화면

### Phase 5: 테스트 및 배포 (6주차)
- 난이도 밸런싱 (공 속도, 파워업 확률 조정)
- 버그 수정 및 예외 처리
- README 및 플레이 가이드 작성
- GitHub 업로드 및 실행 파일 빌드

## 4. 전체 일정(타임라인)

| 주차 | 단계 | 주요 산출물 | 비고 |
|------|------|-------------|------|

**개발 리스크**: 
- 물리 엔진 구현 난이도: 공 반사 각도 계산이 복잡할 수 있으나, PyGame의 기본 충돌 함수 활용 시 완화 가능
- 스테이지 디자인 부담: 초기에는 5~10개 스테이지로 시작하고, 커뮤니티 기여로 확장 가능
- 아날로그 조작감: 마우스 입력으로 부드러운 패들 이동 구현 필요 (키보드는 보조 수단)

**저작권 주의사항**: 원작 Arkanoid는 Taito의 지적재산이나, 게임플레이 메커니즘 자체는 보호 대상이 아님. 다만 "Arkanoid"라는 명칭, 원작 그래픽/사운드는 사용 불가하므로, 프로젝트명은 변경하고 자체 제작 에셋 사용 권장
